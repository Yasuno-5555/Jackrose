# Cidre v0.12.0: macOS Bootstrap Pack

## Summary

v0.12.0 adds the first macOS-side bootstrap entrypoint for Cidre.

## Motivation

The highest-friction point in the current flow is often before Cidre itself starts: users need to get through the ALARM/Asahi installation path first. This release gives Cidre a safe macOS-side entrypoint before the Linux root phase.

## New Commands

- `./install-macos`
- `scripts/cidre-macos-check`
- `scripts/cidre-macos-installer`
- `scripts/cidre-macos-seed`
- `scripts/cidre-macos-handoff`

## New Files

- `docs/macos-install.md`
- `docs/macos-to-cidre-flow.md`
- `docs/installer-threat-model.md`

## Supported Flow

```text
macOS
→ ./install-macos
→ ALARM/Asahi installer
→ fresh ALARM root
→ ./preinstall
→ normal user
→ ./install
```

## Non-goals

- automatic APFS partitioning
- automatic ALARM installation
- automatic rootfs seed injection
- GUI installer
- full firstboot OOBE

## Validation

- shell syntax checks for all new macOS scripts
- Linux-side rejection test for `./install-macos`
- profile validation checks
- static manifest generation path verification

## Known Limitations

- manual continuation is still required after ALARM installation
- the generated seed is recorded for later use but not auto-imported
- macOS dry-run and real-hardware installer handoff still require validation on an Apple Silicon Mac

## Next Steps Toward v0.13.0

v0.13.0 should add seed import and resume support on the ALARM side.
