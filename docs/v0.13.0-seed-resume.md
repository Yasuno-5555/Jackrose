# Cidre v0.13.0: Seed & Resume Pack

## Summary

v0.13.0 connects the macOS bootstrap phase to the fresh ALARM setup phase with seed import and resume support.

## Motivation

v0.12.0 could generate a seed on macOS, but fresh ALARM did not know how to import or use it. v0.13.0 closes that gap.

## New Commands

- `scripts/cidre-seed`
- `scripts/cidre-seed-verify`
- `scripts/cidre-seed-import`
- `scripts/cidre-resume`

## Updated Commands

- `./preinstall --import-seed /path/to/cidre-seed.tar.gz`
- `./install --resume`
- `cidre-doctor --seed`
- `cidre-doctor --resume`

## Seed Format

```text
cidre-seed/
  manifest.json
  resume.env
  checksum.txt
  handoff.txt
  repo-info.txt
```

## Resume Flow

```text
macOS:
  ./install-macos
  -> cidre-seed.tar.gz

fresh ALARM root:
  ./preinstall --import-seed /path/to/cidre-seed.tar.gz

ALARM normal user:
  ./install --resume
```

## Validation

- shell syntax checks
- seed generation and verify checks
- Linux-side non-root import rejection
- installer resume path wiring

## Known Limitations

- manual seed transfer is still required
- rootfs auto-injection is not implemented
- GUI install flow is not implemented

## Next Steps

The next release should reduce package-build friction and move more of the first install into stable package-based flows.
