# Builder Artifacts & Registration

This document outlines the validation, tracking, and local registration of build artifacts generated by the ALARM image builder.

## Artifact Registration

When `cidre-builder-artifacts` runs, it scans configured image output directories for expected target formats (e.g., `.img.xz`, `.tar.gz`, `.raw`).
Registered files are added to the local build manifest (`.local/state/cidre/image-build/cidre-image-manifest.json`).

## Verification Requirements

Before a prototype image can be promoted:
1. **Layout Correctness**: Must contain valid partition schemes or root structure.
2. **Metadata Fields**: Must match checksums and manifest properties.
3. **Boot Readiness Check**: Must have passed the `cidre-image-boot-readiness` criteria.
4. **Promotion Status**: The transition step checks that validation flags (such as boot validation matrix results) are satisfied.
