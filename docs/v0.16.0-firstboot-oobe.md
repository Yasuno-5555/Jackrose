# Cidre v0.16.0: Firstboot OOBE Pack

## Summary

v0.16.0 introduces the firstboot OOBE layer for future Cidre-controlled images.

## Motivation

Cidre needs a first boot path that does not depend on the user already understanding default ALARM credentials and root-phase setup order.

## New Commands

- `scripts/cidre-oobe`
- `scripts/cidre-firstboot-state`
- `scripts/cidre-firstboot-handoff`

## Updated Components

- `scripts/cidre-firstboot-root`
- `scripts/cidre-image-build`
- `scripts/cidre-image-inspect`
- `scripts/cidre-doctor`
- `scripts/cidre-recovery`

## Firstboot State

Firstboot state now records markers, selected user/profile, and handoff output.

## Validation

- firstboot dry-run
- simulated root state transitions
- handoff generation
- overlay sync and inspection
- doctor/recovery integration

## Non-goals

- public bootable image release
- GUI OOBE
- guaranteed real hardware boot

## Known Limitations

- firstboot OOBE is shell/TUI based
- existing ALARM images are unchanged
- root autologin is not enabled by default

## Next Steps

v0.17.0 should move from firstboot scripting toward image boot validation and real firstboot execution checks.
