# Cidre v0.15.0: Cidre Image Prototype Pack

## Summary

v0.15.0 introduces the first prototype image artifact flow for Cidre.

This release does not ship a public image.
It makes it possible to generate, inspect, and validate prototype rootfs overlay artifacts.

## Motivation

Cidre needs an image-level path to remove the gap between a base ALARM image and the first Cidre-controlled boot experience.

v0.15.0 addresses the artifact layer first:

- overlay content
- firstboot-root service presence
- seed/resume tooling presence
- artifact manifest and checksums

## New Scripts

- `scripts/cidre-image-build`
- `scripts/cidre-image-inspect`
- `scripts/cidre-image-checksum`
- `scripts/cidre-image-manifest`
- `scripts/cidre-image-clean`

## New Workspace

- `downstream/image-build/`

## Prototype Artifact Flow

```text
repo scripts + systemd units
↓
overlay sync
↓
rootfs overlay artifact
↓
checksum + manifest
↓
inspection
```

## Validation

- shell syntax checks
- overlay sync checks
- overlay artifact generation
- checksum generation
- manifest generation
- overlay inspection

## Non-goals

- public image release
- production installer metadata integration
- guaranteed boot
- real hardware validation

## Known Limitations

- artifact flow is still prototype-only
- builder integration remains environment-dependent
- firstboot-root remains a prototype

## Next Steps

v0.16.0 should turn the prototype firstboot-root mechanism into a real firstboot OOBE flow.
