# Cidre v0.17.0: Image Boot Validation Pack

## Summary

Cidre v0.17.0 adds image boot validation tooling.

It does not ship a public bootable image.
It provides the scripts needed to verify that a prototype image is ready for controlled boot testing.

## Motivation

v0.15.0 introduced overlay artifact generation.
v0.16.0 introduced firstboot OOBE scripts.

The gap between those and real boot testing was:

- No tooling to inspect a mounted rootfs
- No boot readiness aggregation
- No boot checklist generation
- No structured boot log collection
- No builder environment status check

v0.17.0 fills this gap.

## New scripts

| Script | Purpose |
|---|---|
| `scripts/cidre-builder-status` | Check builder directory and host tool availability |
| `scripts/cidre-image-mount` | Mount a rootfs image read-only |
| `scripts/cidre-image-unmount` | Unmount a mounted rootfs image |
| `scripts/cidre-rootfs-inspect` | Inspect mounted rootfs for Cidre components |
| `scripts/cidre-image-boot-readiness` | Aggregate boot readiness check |
| `scripts/cidre-boot-checklist` | Generate a boot validation checklist |
| `scripts/cidre-boot-log-collect` | Collect post-boot logs |

## Updated scripts

## Non-goals

- Public bootable Cidre image release
- Production ALARM installer metadata integration
- Real Apple Silicon boot success guarantee
- GUI firstboot
- Full desktop auto-install
- Package signing infrastructure
- APFS / boot chain modification

## Known limitations

- v0.17.0 is a boot validation tooling release, not a public bootable image release
- Rootfs inspection does not guarantee boot success
- Image mounting requires root privileges and host tools
- Boot log collection only works after a boot attempt
- Real Apple Silicon boot validation is performed separately on physical hardware
