# Cidre v0.18.0 Builder Integration Pack

This document covers the implementation plan, verification checklist, and system integration details for the Cidre v0.18.0 release.

## 1. 目的

Cidre v0.18.0 の目的は、v0.17.0 までに整備した rootfs overlay / firstboot OOBE / boot validation tooling を、ALARM image builder と接続し、Cidre prototype rootfs image を再現可能な形で生成できるようにすることである。

## 2. 実装スコープ

- **Builder Config Verification (`cidre-builder-config`)**: ビルドディレクトリ、出力ディレクトリ、およびプロファイル名の検証。
- **Overlay Injection (`cidre-builder-integrate`)**: rootfs overlay (`usr/`, `etc/`, `var/`) を builder のステージング領域にコピー・配置する。
- **Builder Invocation Wrap (`cidre-builder-invoke`)**: ビルドの計画と実行の制御。
- **Log Management (`cidre-builder-log`)**: ビルド警告およびエラーの分析。
- **Artifact Register (`cidre-builder-artifacts`)**: 生成されたイメージをスキャンし、ローカルビルドステートのマニフェストへ登録。
- **Promotion Flow (`cidre-image-promote`)**: プロトタイプイメージの配布ディレクトリへの昇格（ブート検証の合格を条件とする）。
