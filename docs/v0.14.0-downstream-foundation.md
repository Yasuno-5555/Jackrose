# Cidre v0.14.0: Downstream Foundation Pack

## Summary

v0.14.0 defines the downstream foundation required to move Cidre toward an ALARM/Asahi image product.

## Motivation

The remaining friction is not only inside Cidre. It is at the boundary where users first boot a base ALARM image and need to know default credentials or next steps.

## New Files

- downstream workspace files
- downstream strategy docs
- image plan and installer integration docs
- firstboot-root prototype and unit examples

## New Scripts

- `scripts/cidre-downstream-check`
- `scripts/cidre-upstream-status`
- `scripts/cidre-image-layout-check`
- `scripts/cidre-installer-metadata-check`
- `scripts/cidre-firstboot-root`

## Non-goals

- public Cidre image shipping
- production installer metadata publishing
- full firstboot OOBE
- package repository rollout

## Validation

- shell syntax checks
- downstream foundation check
- JSON validation for installer entry example
- rootfs overlay layout checks

## Known Limitations

- the image is still a plan, not a shipped artifact
- firstboot-root remains a prototype
- root autologin example is intentionally not installed

## Next Steps

The next release should turn this structure into a real image prototype build path.
