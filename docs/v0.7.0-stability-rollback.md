# Cidre v0.7.0: Stability & Rollback Pack

Cidre v0.7.0 introduces rollback configuration capabilities, user environment snapshots, and a consolidated diagnostics history suite.

## Key Changes

### 1. Unified Configuration Snapshots (`cidre-snapshot`)
- Created the new `cidre-snapshot` utility to take timestamped, user-owned configuration backups before deploying edits.
- Supports commands:
  - `cidre-snapshot create`: Creates configuration recovery points.
  - `cidre-snapshot list`: Displays available recovery snapshots.
  - `cidre-snapshot diff latest`: Diffs files against current state.
  - `cidre-snapshot restore <id|latest>`: Performs rollback restore on Cidre-managed files.
  - `cidre-snapshot prune --keep <num>`: Deletes old snapshots automatically (default limit: 10).

### 2. Pre-apply Configuration Backups (`cidre-user-setup`)
- Updates to `cidre-user-setup` automatically call `cidre-snapshot create` prior to applying configuration profiles.
- Integrated `rollback`, `history`, and `--no-snapshot` parameters.

### 3. Integrated Recovery Snapshot Actions (`cidre-recovery`)
- Extended `cidre-recovery` with `restore latest` and `snapshots` (lists configuration snapshots).
- Split `reset-niri-config` into specific fallback scripts: `reset-niri`, `reset-waybar`, and `reset-shell`.

### 4. Diagnostics Execution Logs & Fix Suggestions (`cidre-doctor`)
- Improved `cidre-doctor` by saving checking states inside `~/.local/state/cidre/doctor/`.
- Introduced arguments:
  - `--last`: Visualizes results from the last diagnostics test execution.
  - `--history`: Visualizes historical execution results.
  - `--fix-suggestions`: Formulates recovery recommendations based on failure indicators.
- Linked daily checks to `cidre-firstboot` graphical validation.
