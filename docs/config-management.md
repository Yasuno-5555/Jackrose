# Cidre Configuration Management Guide

Cidre deploys and manages user configuration profiles safely using the `cidre-user-setup` utility.

## Subcommands Reference

*   **`cidre-user-setup apply [--profile <p>] [--no-snapshot]`**: Installs default configurations to your `$HOME`. Prior to writing files, a configuration snapshot is automatically created (unless `--no-snapshot` is supplied).
*   **`cidre-user-setup dry-run [--profile <p>]`**: Displays what files would be installed, modified, or backed up without writing changes.
*   **`cidre-user-setup status`**: Checks if your home dotfiles are clean (matching defaults) or modified.
*   **`cidre-user-setup diff`**: Shows a unified diff comparing your current home config files against default templates.
*   **`cidre-user-setup rollback [target]`**: Rolls back Cidre configuration using `cidre-snapshot restore` (defaults to latest snapshot).
*   **`cidre-user-setup history`**: Shows execution logs of setup profiles and restore operations.
*   **`cidre-user-setup backup`**: Backs up all managed configuration files.
*   **`cidre-user-setup restore`**: Restores the latest configuration backup.
*   **`cidre-user-setup reset`**: Resets all configuration files to standard templates (creates backup first).

## Managed Profile Levels

*   `minimal`: Installs Niri compositor configurations and fuzzel launcher.
*   `desktop`: Installs Niri, fuzzel, Ghostty shell config, Fcitx5 keyboard, Waybar status layouts, wallpaper, and desktop applications.
*   `developer`: (Default) Installs desktop environment configurations plus Fish shell configs and Starship prompt layouts.
*   `full`: Extends development profiles with extra tools and settings.

## State Directory

All backups, state manifests, and diagnostic reports are managed under:
`~/.local/state/cidre/`
- `manifest.json`: Stores file checksums and installed profile levels.
- `backups/`: Keeps timestamped configuration backups.
- `snapshots/`: Contains configuration snapshots generated by `cidre-snapshot`.
- `doctor/`: Stores diagnostic runs and history metadata.
- `setup/history.log`: Stores configuration modification events.
