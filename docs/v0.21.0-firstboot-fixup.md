# Cidre v0.21.0: Firstboot Fixup Pack

## Summary

Cidre v0.21.0 focuses on stabilizing the first boot initialization path, providing comprehensive tools to diagnose, repair, and retry failures during execution, and improving service boot sequencing.

## Motivation

Firstboot configurations are fragile, often failing due to timing issues or unexpected environment configurations. In previous versions, diagnosing or recovering from firstboot failures required manual marker edits and tedious log inspection. This release introduces automated tools to streamline recovery.

## New Scripts

* `scripts/cidre-firstboot-diagnose`: Checks state directory files, verifying started/completed/failed markers.
* `scripts/cidre-firstboot-retry`: Allows developers to clear failures and schedule retry triggers.
* `scripts/cidre-firstboot-repair`: Fixes missing state dirs, clears failed markers, and regenerates handoffs.
* `scripts/cidre-firstboot-report`: Compiles markdown metrics logs.
* `scripts/cidre-firstboot-console`: Outputs short status outputs to text display.

## Updated Scripts & Services

* `cidre-firstboot-root`: Integrated stage tracking and failure trapping.
* `cidre-oobe`: Styled pure shell fallbacks.
* `cidre-firstboot-state`: Added new state modifier subcommands.
* `cidre-firstboot-handoff`: Added detailed diagnostic and next-step commands.
* `cidre-boot-log-collect`: Added collection hooks for diagnostics and unit definitions.
* `systemd/cidre-firstboot-root.service`: Relaxed startup target requirements.

## Known Limitations

* Stablizes root-phase firstboot but does not guarantee graphical desktop readiness.
* Recovery scripts are conservative and require explicit parameters in certain configurations.
