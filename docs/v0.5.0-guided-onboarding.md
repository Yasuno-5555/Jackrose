# Cidre v0.5.0: Guided Onboarding Pack

Cidre v0.5.0 introduces a guided installation and first-boot flow, connecting all diagnostics, setups, and recovery files into a single, cohesive user journey.

## Key Changes

### 1. Root-Level Entrypoint (`./install`)
- Created a top-level `./install` script that delegates options to the internal orchestration manager.

### 2. Guided Installer Orchestration (`cidre-installer`)
- Implemented `scripts/cidre-installer` to guide the user from preflight system checks, profile choices, dry-run previews, to automated package installation and configuration setup.
- Added non-interactive modes (`--profile <name>`, `--yes`) for automated scripts.
- Log details are written to `~/.local/state/cidre/install.log` on failures.

### 3. Dedicated Installation Profiles
- Map packages and configurations to clear profile targets:
  - `desktop` (Default recommended daily driver)
  - `developer` (Desktop with development tools like fish and starship prompt)
  - `minimal` (Core session config only)
  - `recovery` (Triage recovery commands only)

### 4. First-Boot Audit (`cidre-firstboot` / `cidre-welcome`)
- Created the first-boot diagnostic validator `cidre-firstboot` to verify configuration integrity upon graphical user logins.
- Updated `cidre-welcome` dashboard instructions to print recovery, doctor, and first-boot command states.
