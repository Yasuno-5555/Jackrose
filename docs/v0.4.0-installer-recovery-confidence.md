# Cidre v0.4.0: Installer & Recovery Confidence Pack

Cidre v0.4.0 focuses on strengthening installation, setup validation, preflight diagnostics, and emergency recovery workflows.

## Key Changes

### 1. Hardened Installer Preflight (`bootstrap.sh`)
- Added CLI modes: `--check` (validate required system packages and tools) and `--dry-run` (preview script actions before modifying system files).
- Implemented robust error messaging on script failures.

### 2. Desktop Health Checks (`cidre-doctor`)
- Introduced the `cidre-doctor` tool to print clear health validation check indicators (`OK`, `WARN`, `FAIL`) for window sessions, compositor configurations, Fcitx5 user services, audio safe daemons, and system executables.

### 3. Streamlined Recovery Options (`cidre-recovery`)
- Added the `doctor` subcommand for quick recovery triage checks.
- Added `list` and `restore <file>` subcommands to review and roll back Niri compositor configuration files.
- Provided detailed recovery steps for graphical, IME, and sound failures in [recovery.md](./recovery.md).

### 4. Repository Package Validation
- Created the [check-packages](../scripts/check-packages) script to verify package metadata integrity, validate package relationships, and automatically regenerate AUR `.SRCINFO` files across the Arch package set.
- Ensured zero developer home directories are hardcoded in system configs.

## Validation Status

This release has been statically and sandboxed verified through:
- Bash syntax checks on all scripts
- Python compiler checks for setup managers
- Sandboxed `$HOME` setup configurations deployment tests
- Verification checklist runs
