# Cidre v0.19.0 Real Image Build Pack

This document covers the implementation plan, verification checks, and system integration details for the Cidre v0.19.0 release.

## 1. 目的

Cidre v0.19.0 の目的は、ALARM builder を実際に実行し、Cidre rootfs overlay を組み込んだ prototype rootfs image を生成・登録・mount・inspect できるようにすることである。

## 2. 実装スコープ

- **Environment Diagnostics (`cidre-build-environment`)**: ビルド実行に必要なホストツールの有無や、特権（sudo/losetup 等）の必要性を検査する。
- **Real Builder Exec (`cidre-builder-run`)**: 実イメージ生成用の `build.sh` エントリポイントを呼び出し、結果コードとログを格納する。
- **Artifact Register (`cidre-image-register`)**: 生成されたイメージを検出・選択し、標準ディレクトリ `registered/` への配置と SHA256 ハッシュ生成を行う。
- **Artifact Verification (`cidre-image-verify`)**: 登録されたファイルの存在、チェックサム合致、およびマニフェスト情報の整合性をチェックする。
- **Build Reporting (`cidre-image-build-report` / `cidre-build-failure-report`)**: 成否に関わらず、実行パラメータや失敗トレースを Markdown レポートとして整理する。
- **Orchestration (`cidre-real-image-build`)**: 上記フローを一元管理し、マウント・インスペクト・起動準備状況（boot readiness）チェックまでシームレスに連携させる。

## 3. 非目標

- 本番用 Cidre パブリックイメージの配布
- パッケージ署名インフラの構築
- 実機 Apple Silicon における起動の完全保証
