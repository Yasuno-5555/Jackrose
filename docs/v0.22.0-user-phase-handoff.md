# Cidre v0.22.0: User Phase Handoff Pack

## Summary

Cidre v0.22.0 is a developer stability release focusing on user phase handoff. It introduces machine-readable handoff state serialization, verification tooling for normal user contexts, and safety preflight checks during installer resumption.

## Motivation

Transitioning between root setup and user setup is a common failure point. Normal users need explicit guidance and diagnostic tools to recover if installations fail after login.

## New Scripts

* `scripts/cidre-user-handoff`: Shows, verifies, and imports root phase handoff variables.
* `scripts/cidre-user-phase-state`: Reads and writes user execution states.
* `scripts/cidre-user-phase-verify`: Validates pre-conditions.
* `scripts/cidre-user-phase-report`: Generates markdown diagnostics reports.
* `scripts/cidre-user-phase-repair`: Safely fixes minor state directory omissions.
* `scripts/cidre-first-login`: Displays guided advice.

## Updated Scripts & Services

* `cidre-firstboot-handoff`: Outputs env/json state files to `/var/lib/cidre/handoff/`.
* `cidre-installer`: Added verify hooks, ERR traps, and stage logging.
* `cidre-resume`: Integrated validation subcommands and schema reviews.
* `cidre-doctor`: Added `--user-phase` check options.
* `cidre-recovery`: Added user phase subcommand cases.
* `cidre-boot-log-collect` & `cidre-boot-test-report`: Integrated user phase summaries.
* `cidre-image-build`: Configured script sync list.
