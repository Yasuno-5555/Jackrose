# Cidre v0.6.0: Daily Driver Polish Pack

Cidre v0.6.0 polishes the daily-use desktop experience after guided installation, focusing on niri keybindings, Waybar layouts, sound volume/mute helpers, brightness control, and daily driver diagnostics.

## Key Highlights

### 1. Normalized Niri Session Keybindings
- Changed keybinding to open terminal to `Super + Enter` (Ghostty with foot fallback).
- Configured window fullscreen layout on `Super + F` and maximize column on `Super + Shift + F`.
- Cleaned up confusing, duplicate, and hardcoded bindings in the default `config.kdl`.

### 2. Improved Waybar System State
- Swapped older `pulseaudio` volume module configuration with native `wireplumber` volume controls.
- Configured battery indicator percentage, warning thresholds, charging states, and tooltips.
- Added SSID network status displaying active connection details.

### 3. Extended `cidre-audio` Controls
- Added subcommands: `volume <val>`, `mute`, `unmute`, and `restart` (to restart systemd audio user services).
- Volume controls support relative adjustments (e.g. `5%+`, `5%-`) and mute toggles.

### 4. Sane Screenshot Flow
- Set Print Screen keyboard shortcuts:
  - `Print`: Full screen capture.
  - `Shift + Print`: Region area capture (via `slurp` and `grim`).
  - `Super + Print`: Single active window screenshot.
  - `Ctrl + Print`: Screenshot full screen to clipboard.
  - `Ctrl + Shift + Print`: Area screenshot selection to clipboard.

### 5. Daily Driver Diagnostics (`cidre-doctor --daily`)
- Added new driver verification options checking for `brightnessctl`, `grim`/`slurp` screenshot tools, battery interface availability, and Mozambique IME layout client status.
- Linked daily checks into `cidre-firstboot` graphical audit logs.

## Upgrading

Run `cidre-user-setup --force` to deploy updated defaults, or reload configurations on `Super + Shift + R`.
